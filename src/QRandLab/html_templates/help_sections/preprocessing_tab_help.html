Preprocessing Tab

The Preprocessing tab applies randomness extraction algorithms to improve the quality of random data before statistical testing. It supports multi-file processing with automatic format conversion.

Available Algorithms

  • VonNeuman: Classic Von Neumann extractor. Takes pairs of bits and outputs 0 for "01" and 1 for "10", discarding "00" and "11" pairs. Reduces bias but also reduces output size. Requires STRING01 input.
  • XOR1: XOR folding with adjacent bits. XORs bit n with bit n+1. Reduces output size by half. Requires STRING01 input.
  • XOR2: XOR folding with distant bits. XORs first half of data with second half. Reduces output size by half. Requires STRING01 input.
  • Toeplitz Extractor: Toeplitz matrix-based randomness extractor. Advanced algorithm for extracting near-perfect randomness using Toeplitz hashing. Requires BINARY input and specific parameters (n, l, seed).

File Selection

  • All Files: Process all files currently loaded in the Input tab
  • Single File: Select a specific file from the dropdown to process only that file

Options

  • Output Tag: Text appended to output filenames (e.g., "_vn" produces "data_vn.bin"). Leave empty for no tag.
  • Auto-convert to required format: Automatically convert input files to the format required by the selected algorithm. If unchecked, files with incompatible types will show an error.
  • Convert back to original format: After preprocessing, convert the output back to the original file type. Useful when you want to maintain the same format as input.
  • Add preprocessed files to Input table: Automatically add the processed output files to the Input tab for further operations or testing.

TZ Extractor Settings

The TZ Extractor requires additional configuration (click "TZ Settings" button):

  • n: Input block size parameter (default: 256)
  • l: Output length parameter (default: 128)
  • Output file: Name for the extracted output file
  • TZM file: Name for the TZ matrix file (optional)
  • Save TZM: Whether to save the TZ matrix for analysis

Actions

  • Run: Execute the selected algorithm on chosen files
  • Save: Save processed data to disk (choose output directory)
  • Reset: Clear results and reset all settings to defaults

Format Requirements

Algorithm Required Input Format
VonNeuman STRING01 (text file of 0s and 1s)
XOR1 STRING01
XOR2 STRING01
TZ Extractor BINARY (raw bytes)

Tips

  • Enable "Auto-convert" if your files are not in the required format - the system will handle conversion automatically
  • VonNeuman typically reduces data size by 75% or more, so ensure you have enough input data
  • XOR algorithms reduce size by exactly 50%
  • Use "Convert back to original format" to maintain consistency in your file collection
  • Results are displayed in the Status section with per-file success/failure information
