Report Tab

The Report tab aggregates and displays results from all operations in a unified HTML report. It automatically collects data from file operations, preprocessing, conversions, and statistical tests.

Report Sections

The report is organized into sections that are added as you perform operations:

  • Header: Application title, version, and generation timestamp
  • Input File Information: Details about loaded files
  • File Converter Results: Format conversion operations
  • Preprocessing Results: Randomness extraction algorithm outputs
  • Slicer Results: Data slicing operations
  • ENT Test Results: Entropy analysis with parsed metrics
  • NIST Test Results: Detailed results for each enabled test
  • Dieharder Results: Comprehensive test battery outcomes
  • Borel Test Results: Pattern frequency analysis
  • Autocorrelation Results: Correlation analysis with plot
  • QRNG API Results: Data fetched from online services
  • EYL Device Results: Quantum device generation logs
  • PRNG Results: Pseudo-random generation details

Actions

  • Save Report: Export the complete report as an HTML file. Includes all styling and can be opened in any web browser.
  • Clear Report: Remove all sections and start fresh. Confirmation required - this cannot be undone.
  • Refresh: Update the display with the latest report content

Automatic Data Collection

Results are automatically added to the report when you:

  • Run any statistical test (ENT, NIST, Dieharder, Borel, Autocorrelation)
  • Convert file formats
  • Run preprocessing algorithms
  • Slice data files
  • Fetch data from QRNG APIs
  • Generate data with EYL device or PRNG

Report Features

  • Theme Matching: Report colors automatically match your application theme
  • Formatted Tables: Test results displayed in easy-to-read tables
  • Color-coded Results: PASS (green), FAIL (red), WEAK (orange)
  • Embedded Plots: Autocorrelation graphs embedded directly in report
  • Scrollable View: Navigate through long reports easily

Saving Reports

  • Reports are saved as standalone HTML files
  • All styling is embedded - no external dependencies needed
  • Default filename includes timestamp: QRNG_Report_YYYYMMDD_HHMMSS.html
  • Reports can be opened in any modern web browser
  • Reports can be printed or converted to PDF via browser

Tips

  • Save reports regularly during long testing sessions
  • Clear the report before starting a new analysis to keep results organized
  • Results are preserved until you clear the report or restart the application
