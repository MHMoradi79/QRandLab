Input Tab

The Input tab is your starting point for working with files in QRandLab. It allows you to import, manage, and validate multiple files for randomness testing and processing.

File Table

The main area displays all loaded files in a table with the following columns:

  • Name: The file name without extension
  • Folder: The directory path where the file is located
  • Ext: The file extension (.bin, .txt, .dat, etc.)
  • Size: File size in human-readable format (KB, MB, etc.)
  • Type: The data format type (binary, string01, hex, uint8, etc.)

File Types

Understanding file types is crucial for proper testing. Each type represents how data is stored:

  • binary: Raw binary data (bytes). Used by ENT and Dieharder tests.
  • string01: Text file containing only '0' and '1' characters. Used by NIST, Borel, and Autocorrelation tests.
  • hex: Hexadecimal text representation of data
  • uint8: Unsigned 8-bit integers (0-255) in text format
  • uint16: Unsigned 16-bit integers (0-65535) in text format
  • uint32: Unsigned 32-bit integers in text format
  • uint64: Unsigned 64-bit integers in text format

Action Buttons

  • + Add Files: Opens a file dialog to select one or more files to import
  • - Remove: Removes selected files from the table (does not delete actual files)
  • Validate Files: Checks all files to ensure their content matches their declared type
  • Restart: Restarts the entire application
  • Reset: Clears all files and resets the tab to its initial state

Context Menu (Right-Click)

Right-clicking on a file row shows additional options:

  • Open Folder: Opens the file's directory in your system file explorer
  • Open File: Opens the file with your system's default application
  • Rename: Rename the file on disk
  • Read Sample: View a sample of the file's contents (first N bytes)
  • Set Type: Manually change the file's data type (options depend on extension)

Type Assignment Rules

File types are assigned based on extension:

  • .bin, .dat files: Automatically set to "binary" type
  • .txt files: Can be any text-based type (string01, hex, uint8, etc.). The system attempts to auto-detect the format.

Status Section

The status area at the bottom displays operation results, validation messages, and any errors that occur during file operations.

Tips

  • Double-click a file row to quickly change its type
  • Use Validate Files before running tests to ensure data integrity
  • Hover over a row to see the full file path in a tooltip
  • You can select multiple files by holding Ctrl or Shift while clicking
