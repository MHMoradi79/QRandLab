Converter Tab

The Converter tab transforms random data files between different formats. It supports batch conversion of multiple files with consistent settings.

Supported Formats

  • binary: Raw binary data (bytes). Most compact format. Used by ENT and Dieharder tests.
  • string01: Text file containing only '0' and '1' characters. Required by NIST, Borel, and Autocorrelation tests.
  • hex: Hexadecimal text representation (e.g., "4F A3 2B...")
  • uint8: Space-separated 8-bit unsigned integers (0-255)
  • uint16: Space-separated 16-bit unsigned integers (0-65535)
  • uint32: Space-separated 32-bit unsigned integers
  • uint64: Space-separated 64-bit unsigned integers

File Selection

  • All Files: Convert all files loaded in the Input tab
  • Single File: Select a specific file to convert individually
  • Source Types: Displays the format(s) of currently selected files

Conversion Options

  • Convert To: Target format for conversion
  • Output Tag: Text appended to output filenames (e.g., "_hex" produces "data_hex.txt"). Leave empty for no tag.
  • Add converted files to Input table: Automatically import the converted files into the Input tab for further processing

Actions

  • Convert: Execute the format conversion on selected files
  • Save: Save converted files to disk (choose output directory)
  • Reset: Clear results and reset all settings to defaults

Conversion Matrix

All formats can be converted to any other format. Common conversions:

From To Use Case
binary string01 Prepare for NIST/Borel testing
string01 binary Prepare for ENT/Dieharder testing
hex binary Convert hex dumps to raw bytes
uint8 binary Convert integer lists to raw bytes

Output Files

  • Text-based formats (string01, hex, uint*) are saved with .txt extension
  • Binary format is saved with .bin extension
  • Original files are never modified - new files are created

Tips

  • Converting to string01 increases file size significantly (8x for binary to string01)
  • Converting to binary produces the most compact files
  • Use "Add to Input table" to chain conversions with other operations
  • The status section shows per-file conversion results with success/failure indicators
