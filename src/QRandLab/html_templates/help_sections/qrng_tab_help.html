QRNG Tab

The QRNG (Quantum Random Number Generator) tab provides access to various sources of random data. It contains three inner tabs: Internet APIs, EYL Device, and PRNG.

Internet APIs

Fetch quantum random data from online QRNG services. Three providers are supported:

Available Providers

  • ANU (Australian National University): Quantum random numbers from vacuum fluctuations. Requires API key. Max length: 1024. Supports uint8, uint16, hex16 data types.
  • Random.org: True random numbers from atmospheric noise. Requires API key. Max length: 10000. Supports uint8 data type.
  • HotBits: Random numbers from radioactive decay. Free service with limits. Max length: 759. Supports uint8 data type.

API Settings

  • Length: Number of random values to fetch (provider-specific limits apply)
  • Data Type: Format of returned data (uint8, uint16, hex16 - availability varies by provider)
  • Block Size: For ANU only when using hex16. Valid range: 1-10. Disabled for other data types.
  • API Key: Required for ANU and Random.org. Not needed for HotBits.

Actions

  • Fetch: Request random data from the selected provider
  • Save: Save fetched data to a file (enabled after successful fetch)
  • Reset: Clear status and reset settings to defaults

EYL Device

Interface for the EYL quantum random number generator hardware device. Requires the EYL DLL to be properly installed.

Generation Modes

  • Offline: Generate files locally without streaming. Specify filename, number of files, and size per file.
  • Display: View incoming random data in real-time. Optionally save to file.
  • Stream: Send random data over network (TCP). Can stream to localhost or custom LAN address.

Offline Mode Settings

  • Base Filename: Name prefix for generated files (e.g., "eyl_data" produces eyl_data_001.bin)
  • Number of Files: How many files to generate
  • File Size (bits): Size of each file in bits
  • Output Format: Data format (binary, string01, hex, uint8, uint16, uint32, uint64)

Stream Mode Settings

  • Buffer Size: Bytes per read request (max: 16384)
  • Pause Time: Milliseconds between device requests (0-10000). Use higher values to avoid -4 errors on high buffer sizes.
  • Network: localhost or LAN (custom)
  • Host/Port: Network address for streaming (default: 127.0.0.1:4000)
  • Save sent data: Optionally save streamed data to file

Connection

  • Check DLL: Verify EYL driver is installed and accessible
  • Connect to EYL: Toggle device connection (enabled after DLL check passes)

Save Options

  • Save Folder: Directory where files will be saved
  • Base Name: Filename prefix for saved data
  • Add Timestamp: Include date/time in filename

PRNG (Pseudo-Random Number Generator)

Generate pseudo-random data using the DieHarder library's PRNG algorithms. Useful for testing and comparison with true random sources.

DieHarder PRNG Settings

  • Generator ID: Algorithm identifier. Valid ranges: 0-61, 203-206, 400-405, 500-501
  • Seed: Initial seed value for the generator (same seed produces same sequence)
  • Number of Files: How many files to generate (max: 1000)
  • Size per File: Bytes per file (max: 1GB)

Common Generator IDs

  • 0: Borosh13
  • 1: CMRG (Combined Multiple Recursive Generator)
  • 13: Mersenne Twister (mt19937)
  • 14: MT19937-1999
  • 22: RANLUX (luxury level algorithms)

Saving Options

  • Include Header: Add generator info at the start of each file
  • Generator ID in filename: Include the algorithm ID in the filename
  • Seed in filename: Include the seed value in the filename
  • Base name: Filename prefix (default: "prng")

Actions

  • Start: Begin generating PRNG data
  • Stop: Cancel ongoing generation
  • Save Path: Choose output directory
  • Reset: Clear status and reset to defaults

Tips

  • PRNG data is deterministic - the same seed and generator always produce identical output
  • Use PRNG for baseline testing or when you need reproducible "random" data
  • Files are saved directly to disk as they are generated (no need for separate save step)
