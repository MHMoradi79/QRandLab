Tests Tab

The Tests tab provides access to various statistical randomness tests. Each test evaluates different aspects of data quality and can run on single or multiple files.

File Selection

  • All Files: Run the selected test on all loaded files
  • Single File: Select a specific file to test
  • Auto-convert to required format: Automatically convert files to the format required by each test

ENT Test

Entropy analysis using the external ENT tool. Provides quick statistical analysis of randomness quality.

Required Format: binary

ENT Settings

  • Bit stream mode: Analyze data as individual bits instead of bytes
  • Occurrence count: Include byte/bit frequency distribution
  • Fold: Fold uppercase letters to lowercase before analysis
  • CSV output: Generate output in comma-separated format
  • Save raw output: Save the raw ENT output to a file

ENT Results

  • Entropy: Bits of information per byte (max 8.0 for bytes, 1.0 for bits)
  • Chi-square: Measures deviation from expected distribution
  • Mean: Average value (ideal: 127.5 for bytes, 0.5 for bits)
  • Monte Carlo Pi: Pi estimation using random coordinates
  • Serial Correlation: Measures relationship between consecutive values

NIST Statistical Test Suite

Comprehensive battery of 16 tests from NIST SP800-22 for evaluating cryptographic random number generators.

Required Format: string01

NIST Settings

  • Enabled Tests: Select which of the 16 tests to run
  • Sequence Length: Number of bits per test sequence (default: 1,000,000)
  • Use Whole Data: Use entire file instead of specified sequence length
  • P-value Threshold: Significance level for pass/fail (default: 0.01)

NIST Tests Available

  • 1. Frequency: Proportion of 0s and 1s
  • 2. Block Frequency: Proportion within blocks
  • 3. Runs: Oscillation between 0s and 1s
  • 4. Longest Run: Longest run of 1s in blocks
  • 5. Rank: Matrix rank test
  • 6. FFT: Spectral test (frequency domain)
  • 7. Non-overlapping Template: Aperiodic pattern matching
  • 8. Overlapping Template: Periodic pattern matching
  • 9. Universal: Compressibility test
  • 10. Linear Complexity: LFSR complexity
  • 11. Serial: Frequency of patterns
  • 12. Approximate Entropy: Regularity measure
  • 13. Cumulative Sums: Random walk deviation
  • 14. Random Excursions: Random walk returns
  • 15. Random Excursions Variant: Random walk visits

Dieharder Test Suite

Comprehensive battery of 32 randomness tests. Very thorough but time-consuming (30-60 minutes for all tests).

Required Format: binary

Dieharder Settings

  • Test Mode: Run all 32 tests or select specific ones
  • P-samples: Number of p-values to generate (default: 100)
  • T-entities: Number of test entities (default: 10000)
  • KS Mode: Kolmogorov-Smirnov test mode
  • Weak Threshold: P-value below which result is "WEAK"
  • Fail Threshold: P-value below which result is "FAIL"
  • Test Strategy: Standard, Resolve Ambiguity, or Test to Destruction

Borel Normality Test

Tests for Borel normality - whether all possible patterns appear with expected frequency.

Required Format: string01

Borel Settings

  • Min Pattern Length: Minimum pattern size to test (default: 1)
  • Max Pattern Length: Maximum pattern size to test (default: 10)
  • Auto Mode: Automatically determine maximum based on data size

Borel Results

For each pattern length, reports chi-square ratio and fraction ratio. Overall PASS/FAIL based on whether all pattern lengths pass.

Autocorrelation Test

Analyzes correlation between bits at different lag distances. Ideal random data shows no correlation.

Required Format: string01

Autocorrelation Settings

  • Max Lag: Maximum lag distance to compute (default: 100)
  • Use Whole Data: Use all data instead of a sample

Autocorrelation Results

Generates a plot showing correlation coefficients at each lag. Values should be close to zero for random data.

Tips

  • Enable "Auto-convert" if your files are not in the required format
  • Start with faster tests (ENT, Borel) before running lengthy ones (Dieharder)
  • Results are automatically added to the Report tab for export
  • Multi-file tests show per-file results in the status section
