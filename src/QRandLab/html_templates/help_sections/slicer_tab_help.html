Slicer Tab

The Slicer tab extracts portions of data files for focused testing or analysis. It supports multiple slicing operations on single or multiple files.

Slicing Operations

  • Single Slice: Extract a continuous portion of data from a specific position
  • Equal Parts: Divide file(s) into N equal-sized segments
  • Random Sampling: Extract random samples from the data

Single Slice Parameters

  • Start Position (bytes): Where to begin extraction (0 = start of file)
  • Length (bytes): Number of bytes to extract

Example: Start=1000, Length=5000 extracts bytes 1000-5999 from the file.

Equal Parts Parameters

  • Number of Parts: How many equal segments to create (e.g., 4 parts from a 1MB file creates four 256KB files)

Useful for testing different portions of large files independently.

Random Sampling Parameters

  • Sample Size (bytes): Size of each random sample
  • Number of Samples: How many random samples to extract
  • Seed (optional): Random seed for reproducible sampling

Samples are extracted from random positions within the file.

File Selection

  • All Files: Apply slicing operation to all loaded files
  • Single File: Select a specific file (shows file size for reference)

Options

  • Output Tag: Text appended to output filenames (e.g., "_slice" produces "data_slice_001.bin"). Leave empty for no tag.
  • Add sliced files to Input table: Automatically add extracted slices to the Input tab for testing

Actions

  • Run: Execute the slicing operation
  • Save: Save sliced data to disk (choose output directory)
  • Reset: Clear results and reset all settings to defaults

Output Naming

Output files are named based on the operation:

  • Single Slice: filename_tag.ext
  • Equal Parts: filename_tag_001.ext, filename_tag_002.ext, etc.
  • Random Sampling: filename_tag_sample001.ext, etc.

Tips

  • Use Single Slice to extract specific regions of interest from large files
  • Equal Parts is useful for comparing randomness across different sections of a file
  • Random Sampling helps create representative subsets for quick testing
  • File type is preserved during slicing (binary stays binary, string01 stays string01)
  • The status section displays per-file results with the size of each extracted slice
